security-demo

REST API Security with WildFly Elytron & Jakarta Security annotations.

Public (no auth)

  • GET /api/public — Health check (no login)

Secured (BASIC auth)

  • GET /api/secured/whoami — Show user identity + roles
  • GET /api/secured/user-area — @RolesAllowed("user", "admin")
  • GET /api/secured/admin-area — @RolesAllowed("admin") only
  • GET /api/secured/dashboard — Different response per role
  • GET /api/secured/maintenance — @DenyAll (always 403)
Setup required: Add users to WildFly before testing: 
add-user.bat -a -u alice -p alice123 -g user
add-user.bat -a -u bob   -p bob123   -g user,admin
Test: Use IntelliJ HTTP Client with Authorization: Basic ... header, or open secured URLs in browser (login dialog).
